O nome "SWR" é derivado de "stale-while-revalidate", uma estratégia de invalidação de cache HTTP popularizada por 
[HTTP RFC 5861](https://tools.ietf.org/html/rfc5861). SWR é uma estratégia para primeiro retornar os dados do cache 
(estale), depois enviar a requisição de busca (revalidate), e finalmente trazer os dados atualizados.

<Callout emoji="✅">
  Com o SWR, os componentes receberão um fluxo de atualizações de dados <strong>constante</strong> e 
  <strong>automaticamente</strong>.<br/>E a IU será sempre <strong>rápida</strong> e <strong>reativa</strong>.
</Callout>

## Visão Geral

```jsx
import useSWR from 'swr'

function Profile() {
  const { data, error } = useSWR('/api/user', fetcher)

  if (error) return <div>failed to load</div>
  if (!data) return <div>loading...</div>
  return <div>hello {data.name}!</div>
}
```

Neste exemplo, o hook `useSWR` aceita uma chave `string` e uma função `fetcher`. `key` é um identificador único dos 
dados (normalmente a URL da API) e vai ser passado para o `fetcher`. `fetcher` pode ser uma função assíncrona que 
retorna os dados, voce pode usar o fetch nativo ou uma ferramenta como Axios. O hook retorna 2 valores: 
`data` e `error`, baseado no status da requisição.

## Recursos

Com apenas uma linha de código, você pode simplificar a lógica de busca de dados em seu projeto, 
e também ter todas essas maravilhas de funcionalidades fora-da-caixa:

- Busca de dados **rápida**, **leve** e **reusável**
- **Cache** integrado e desduplicação de solicitação
- Experiência em **tempo real**
- Transporte e protocolo agnóstico
- Suporte a SSR / ISR / SSG
- Pronto para TypeScript
- React Native

SWR cobre todos os aspectos de velocidade, exatidão e estabilidade para ajudá-lo a construir melhores experiências:

- Rápida navegação de página
- Polling com intervalo
- Dependência de dados
- Revalidação ao foco
- Revalidação na recuperação da rede
- Mutação local (UI otimista)
- Tentativa inteligente quanto com error
- Paginação e recuperação de posição de rolagem
- React Suspense

E muito [mais](/docs/getting-started).
